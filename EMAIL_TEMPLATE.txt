## 🎓 Key Concepts

### RAG (Retrieval-Augmented Generation)
The application uses RAG to ground LLM responses in actual documents:
1. User uploads PDF
2. Document is chunked and embedded
3. User query is embedded
4. Similar chunks retrieved from vector DB
5. LLM generates response based on retrieved chunks only
6. No hallucinations - only facts from the document

### Zero Hallucination Policy
If information is not in the document, the response explicitly states: *"This information is not available in the document."*
